# Agent Diary

Single-file static HTML visualizer for a Paperclip company's recent activity.
Renders issues, comments, parent/child grouping, and blocker dependencies as
a vertical timeline you can scan in under a minute.

## One-command run

From the repo root:

```bash
npm install
npm run diary -- --company BUI --days 7
open artifacts/agent-diary/index.html
```

`open` works on macOS; on Linux use `xdg-open`. The visualizer is a single
HTML file with inline CSS + JS plus a sibling `data.js` shim, and works when
opened directly from disk (`file://`). No server required.

## Files

- `index.html` — the visualizer. Loads `data.js` (preferred, works from
  `file://`) and falls back to `fetch('./data.json')` if served over HTTP.
- `data.json` / `data.js` — generated by `npm run diary`. Not committed.
- `../../scripts/diary-fetch.ts` — the Node + TypeScript fetch script.

## Env

Both vars are auto-injected during a Paperclip heartbeat run. Outside one,
set them explicitly:

- `PAPERCLIP_API_URL` — e.g. `http://127.0.0.1:3100`
- `PAPERCLIP_API_KEY` — bearer token for the API
- `PAPERCLIP_COMPANY_ID` — UUID of the company you can read

`--company <PREFIX>` must match the issue prefix of the company that
`PAPERCLIP_COMPANY_ID` resolves to. Mismatch is a hard error.
